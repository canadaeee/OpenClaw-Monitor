# 监测指标与页面文档 - OpenClaw Monitor

## 1. 核心监测指标

### 1.1 Agent 状态监控
- **在线状态**：
  - 显示每个 agent 是否在线。
  - 如果 agent 超过设定时间未活动，则标记为 **offline**。
  - **最近活动时间**：记录 agent 最后一条活动的时间戳。
- **任务状态**：
  - 显示当前 agent 正在执行的任务，任务状态分为：
    - **运行中**：任务正在执行。
    - **等待中**：任务正在等待资源。
    - **空闲**：任务已完成或无任务待处理。
    - **失败**：任务执行失败。

### 1.2 任务流转监控
- **任务 ID**：唯一标识任务。
- **任务来源**：任务发起的 agent。
- **当前执行 agent**：当前处理该任务的 agent。
- **任务状态**：
  - **queued**：任务已接收，等待执行。
  - **running**：任务正在执行。
  - **waiting**：任务等待中（例如等待 node 执行）。
  - **failed**：任务执行失败。
  - **done**：任务已完成。
- **token 消耗**：
  - **输入 token**：传递给模型的文本长度。
  - **输出 token**：模型生成的响应文本长度。
  - **总 token**：输入 token 和输出 token 之和。
  
### 1.3 Token 消耗统计
- **按模型分类**：
  - 按模型展示每个 agent 使用的模型类型（如 GPT-3.5、GPT-4、Codex）及其 token 消耗情况。
  - **输入 token** 和 **输出 token** 进行分开统计，并展示 **总 token 消耗**。
  
- **按供应商分类**：
  - 记录模型的供应商（如 OpenAI、Google、Anthropic），按供应商展示每个供应商的 token 消耗。

- **按时间维度**：
  - 按日、周、月查看每个模型和供应商的 **token 消耗趋势**。

### 1.4 节点状态监控
- **Node 在线/离线**：监控 Node 是否在线。
- **CPU/内存占用**（可选）：记录 Node 上的资源使用情况。

## 2. 页面结构与展示内容

### 2.1 总览页
- **显示系统健康状况**：
  - 在线 agent 数
  - 在线 node 数
  - 运行任务数
  - 今日 token 消耗
  - 今日错误数

### 2.2 Agent 页
- **每个 agent 的详细状态**：
  - 当前任务
  - 今日 token 消耗
  - 最近错误信息

### 2.3 任务页
- **每个任务的详细信息**：
  - 任务 ID
  - 来源 agent
  - 当前执行 agent
  - 状态（运行中/等待/失败/完成）
  - 耗时
  - token 消耗
  - 产物路径

### 2.4 Token / Cost 页
- **展示 token 消耗分类视图**：
  - 按模型、供应商分类展示 token 消耗情况
  - 支持按时间维度（按天、按周、按月）查看 token 消耗趋势。

### 2.5 事件流页
- **展示所有事件日志**：
  - 任务状态变化、模型调用、token 消耗等。

### 2.6 Error / Alert 页
- **显示告警信息**：
  - 超时任务
  - 失败任务
  - 异常 token 消耗等
