# 前端页面规格文档 - OpenClaw Monitor

## 1. 目标

本文件用于补充前端当前已确认的页面结构、展示模块和字段需求，作为后端接口实现时的直接参考。

默认约束：

- 左侧导航固定，不随页面切换改变结构。
- 系统默认语言为中文。
- 支持主题模式：跟随系统 / 浅色 / 深色。
- 页面风格统一沿用当前深色控制台式布局。
- Monitor 独立开发，不要求与 OpenClaw 本体使用相同语言。
- 当前前端技术栈固定为 React。

## 2. 全局布局

### 2.1 左侧固定区

包含以下固定内容：

- 产品标题：`OpenClaw Monitor`
- 副标题：`Openclaw可视化监控台`
- 导航项：
  - 总览
  - Agent
  - 任务
  - Token / 成本
  - 事件流
  - 告警
- 网关显示：
  - `Ubuntu Gateway`
  - `Windows Gateway`
  - `Mac Gateway`
  - `Gateway`
- 主题切换
- 语言显示

### 2.2 右侧顶部区

每个页面共用：

- 页面标题
- 页面描述
- 数据连接状态
- 默认端口按钮
- 快照时间按钮

## 3. 页面规格

## 3.1 总览页

### 模块

- Hero 横幅
- 四张统计卡
- 任务追踪
- Agent 状态
- 活跃告警
- 健康摘要

### 字段需求

#### Hero 横幅

- `gatewayLabel`
- `overview.dataLatencySeconds`
- `overview.openAlertCount`
- `overview.taskFailedCount`

#### 统计卡

- `overview.agentOnlineCount`
- `overview.agentOfflineCount`
- `overview.nodeOnlineCount`
- `overview.nodeOfflineCount`
- `overview.taskRunningCount`
- `overview.taskWaitingCount`
- `overview.todayTokenTotal`
- `overview.todayEstimatedCost`

#### 任务追踪

- `task.id`
- `task.taskType`
- `task.executorNodeId`
- `task.tokenUsage.total`
- `task.durationMs`
- `task.status`

#### Agent 状态

- `agent.id`
- `agent.name`
- `agent.connectivityStatus`
- `agent.runtimeStatus`
- `agent.derivedStatusReason`

#### 活跃告警

- `alert.id`
- `alert.title`
- `alert.description`
- `alert.severity`
- `alert.triggeredAt`

#### 健康摘要

- `overview.agentOnlineCount`
- `overview.agentOfflineCount`
- `overview.taskWaitingCount`
- `overview.todayEstimatedCost`
- `overview.todayErrorCount`

## 4. 前端派生字段

以下字段目前可由前端派生，不要求后端单独返回：

- 在线率
- 等待压力描述
- 文案级状态标签
- 网关展示文案

## 5. 建议后端补充的视图模型

为了减少前端拼装复杂度，建议后端后续提供以下视图接口：

- `GET /api/overview`
- `GET /api/agents`
- `GET /api/tasks`
- `GET /api/alerts`
- `GET /api/events`

其中 `events` 建议返回已经适合页面展示的摘要字段，而不仅仅是原始 payload。
